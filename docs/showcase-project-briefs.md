# Showcase Project Briefs

All pages share only the base product task: create a standalone Muse AI Campaign Studio page for a creative director. Each showcase must make its own decisions about layout, visual system, copy rhythm, density, interaction emphasis, and component vocabulary.

## 01 Standard Builder

- Requirement analysis: establish the neutral baseline for a working product screen.
- Design breakdown: conventional app shell, clear form, central preview, right metrics, minimal styling.
- Must differ by: predictable product structure and low visual risk.

## 02 Visual Frontend

- Requirement analysis: prove the visual skill can turn the same task into a memorable visual surface.
- Design breakdown: poster-led layout, high contrast, large composition, fewer panels, stronger image-like preview.
- Must differ by: visual-first hierarchy, dramatic hero canvas, strong color blocking.

## 03 Design Logic

- Requirement analysis: show information hierarchy and layout reasoning more than decorative styling.
- Design breakdown: strict editorial grid, ruled columns, sequence-based decisions, typography as structure.
- Must differ by: asymmetric layout, visible grid logic, deliberate ordering.

## 04 Impeccable Full Flow

- Requirement analysis: present a polished end-to-end product flow with complete states.
- Design breakdown: refined dark product UI, stateful header, clear save/export path, quiet panels.
- Must differ by: quality of details, spacing, contrast, state coverage.

## 05 Artifact Builder

- Requirement analysis: show a self-contained artifact surface, like a bundled single-page deliverable.
- Design breakdown: code/workbench metaphor, file panel, build output, artifact preview, export readiness.
- Must differ by: bundled artifact language and terminal-like composition.

## 06 UX Pro Reference

- Requirement analysis: prioritize usability, accessibility, path clarity, error and loading states.
- Design breakdown: task flow map, form validation, explicit step path, status-driven right rail.
- Must differ by: state coverage and affordance clarity.

## 07 Component System

- Requirement analysis: show component consistency and design system behavior.
- Design breakdown: component inventory, tokens, variants, table-like details, structured system preview.
- Must differ by: reusable component vocabulary and consistent state styling.

## 08 Motion Bits

- Requirement analysis: show how motion and microinteractions improve the generated campaign workflow.
- Design breakdown: motion controls, animated tracks, version transitions, loading feedback.
- Must differ by: visible motion language and interaction feedback.

## 09 Standard + Taste

- Requirement analysis: correct baseline AI aesthetics without making the page decorative.
- Design breakdown: restrained layout, edited copy, calmer palette, deliberate spacing.
- Must differ by: removal of generic AI visual habits.

## 10 Standard + Impeccable

- Requirement analysis: keep the baseline product structure but refine it to production quality.
- Design breakdown: dark refined shell, stronger state model, better export/save hierarchy.
- Must differ by: same product logic, better finish.

## 11 Visual + Taste

- Requirement analysis: keep a visual-led page while removing excess and making it feel expensive.
- Design breakdown: large visual field, quieter palette, fewer widgets, better rhythm.
- Must differ by: visual power with restraint.

## 12 Visual + Impeccable

- Requirement analysis: push strong visual work to a polished high-end result.
- Design breakdown: cinematic split surface, crisp side inspector, confident interaction states.
- Must differ by: high visual impact plus clear product controls.

## 13 Design + UX Pro

- Requirement analysis: combine layout logic and UX completeness.
- Design breakdown: blueprint workflow, form-first left rail, step path, evidence-oriented preview.
- Must differ by: structure plus usability.

## 14 Design + Impeccable

- Requirement analysis: take rational layout and polish it into a sharp editorial product tool.
- Design breakdown: strict columns, refined typography, precise contrast, edited modules.
- Must differ by: hard grid with premium finish.

## 15 Balanced Chain

- Requirement analysis: show the general-purpose chain that works for most product pages.
- Design breakdown: balanced density, clear form, preview, variants, metrics, timeline.
- Must differ by: no single extreme, stable general quality.

## 16 Visual Premium Chain

- Requirement analysis: showcase the best visual-display chain.
- Design breakdown: immersive visual canvas, compact inspector, premium motion feel.
- Must differ by: strongest visual stage while still operating as a product page.

## 17 Product Polish Chain

- Requirement analysis: make the page feel like a real shipped product frontend.
- Design breakdown: navigation, componentized workflow, responsive mode switch, production status.
- Must differ by: product engineering clarity and interface guidelines.

## 18 Max Quality Chain

- Requirement analysis: combine design logic, UX, guidelines, and polish into the highest-complexity page.
- Design breakdown: dense quality cockpit, full side rails, quality stack, metrics, activity, complete local state.
- Must differ by: most complete product UI and strongest overall control surface.
